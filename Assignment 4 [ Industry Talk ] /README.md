### Summary of what i gained

### 1. ICT Is About Systems, Not Just Code
- ICT professionals build **financial systems that handle real money and real users**.
- In fintech and Web3, mistakes directly affect **security, trust, and business continuity**.

### 2. Real Expectations of an Android Developer
An Android Developer in industry is expected to:
- Develop and maintain mobile applications beyond UI design.
- Integrate applications with backend services and blockchain systems.
- Monitor crashes, performance, and user behaviour.
- Continuously improve app reliability and security.

**Common tools used in industry:**
- Android Studio
- Firebase
- Postman
- Bitbucket

These tools are **baseline requirements**, not advanced skills.

### 3. Agile/SCRUM Is a Discipline
- Development is done in short sprints with frequent reviews.
- Daily stand-ups enforce accountability and communication.
- Teams continuously plan, review, and improve their work.
This approach enables faster delivery and adaptability in fast-changing industries like fintech.

### Suggestion for improvement

To further strengthen the session:
- Include **real-world failure or incident case studies** (e.g., security breaches, system downtime).
- Provide a **clear technical learning roadmap** for students.
- Add **technical demonstrations** showing how applications interact with backend or blockchain systems.
- Emphasise **self-learning and continuous upskilling**, as industry evolves faster than university syllabi.

---

### Conclusion

This industry talk highlighted that ICT is a **responsibility-driven profession**. Developers are not just problem-solvers, but custodians of systems that users and businesses depend on daily.

The session shifted my mindset from learning for assessments to learning for **real-world impact**. To be industry-ready, I must continuously build skills, understand systems holistically, and stay disciplined in improving both technical and professional competencies.
